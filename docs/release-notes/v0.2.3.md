# OpenClerk v0.2.3

This patch release carries post-v0.2.2 runner, skill, eval, and release-procedure hardening, with expanded repo-docs dogfooding promoted to mandatory pre-release evidence.

## Changed

- Added runner-owned public web/HTML source intake through `ingest_source_url` with `source.source_type: "web"`, while keeping browser automation, login, purchase, captcha, paywall, direct HTTP/MCP bypass, and direct vault acquisition outside the supported surface.
- Enriched `ingest_source_url` update responses with normalized source URL, source type, source document identity, previous/new SHA evidence, changed/no-op status, duplicate status, stale dependents, projection refs, provenance refs, no-repair warnings, and synthesis repair status.
- Added read-side `tag` filter support to `list_documents` and retrieval `search` as sugar over canonical markdown/frontmatter metadata, without replacing markdown as authority.
- Added the read-only retrieval `memory_router_recall_report` action for evidence-backed recall/routing reports with canonical evidence refs, provenance refs, synthesis freshness, validation boundaries, and authority limits, without promoting a memory transport, `remember`/`recall` API, autonomous router API, embeddings/vector DB, graph memory, or hidden authority ranking.
- Hardened the OpenClerk skill policy for low-risk note capture, duplicate candidate checks, document-these-links placement, opaque artifact boundaries, supplied transcript media intake, local-file-derived duplicate/provenance checks, safe parallel reads, and configuration diagnostics.
- Expanded repo-docs dogfooding so committed public OpenClerk markdown is imported into an isolated eval vault and exercised for retrieval, source-linked synthesis, decision records, release-readiness answers, tag filters, memory-router recall reports, and release synthesis freshness.
- Documented repo-docs dogfooding as a mandatory pre-release step in the maintainer and release-verification procedures while keeping it separate from the full AgentOps production gate.
- Documented `ingest_source_url` update mode: missing `source.mode` defaults to `create`, duplicate creates reject, `source.mode: "update"` re-ingests an existing `source.url`, and mismatched `path_hint` or `asset_path_hint` values conflict without writing extra docs or assets.
- Verified source URL update freshness behavior: same-SHA updates are no-ops without `source_updated` provenance or projection invalidation churn, while changed-PDF updates preserve source and asset paths, refresh searchable extracted text and citations, emit previous/new SHA provenance, and expose stale dependent synthesis through `projection_states`.
