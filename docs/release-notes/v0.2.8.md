# v0.2.8 Release Notes

This patch release adds a general `openclerk config` runner domain for
persisted product/profile configuration while keeping provider settings under
`openclerk module`.

- Added `inspect_config`, `configure_profile`, and `clear_profile` actions for
  the default persisted autonomy profile.
- Applied persisted profile defaults to `openclerk document` and
  `openclerk retrieval`, with request-level `autonomy` fields overriding those
  defaults field-by-field.
- Added CLI, runner, and persistence tests for profile validation, write gates,
  strict JSON handling, and persistence across invocations.
