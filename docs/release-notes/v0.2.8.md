# OpenClerk v0.2.8

This patch release adds a general `openclerk config` runner domain for persisted product/profile configuration while keeping provider settings under `openclerk module`.

## Changed

- Added `inspect_config`, `configure_profile`, and `clear_profile` actions for the default persisted autonomy profile.
- Expanded `inspect_config` into the read-only effective config summary for storage, profile defaults, non-secret module summaries, and git lifecycle checkpoint gate posture.
- Documented that vault-root binding remains under `openclerk init --vault-root`; `inspect_config.storage` reports it but `openclerk config` does not rebind it.
- Applied persisted profile defaults to `openclerk document` and `openclerk retrieval`, with request-level `autonomy` fields overriding those defaults field-by-field.
- Kept git checkpoint enablement invocation-scoped through `--git-checkpoints` or `OPENCLERK_GIT_CHECKPOINTS`; checkpoint persistence is reported as unsupported.
- Added CLI, runner, and persistence tests for profile validation, write gates, strict JSON handling, and persistence across invocations.
- Preserved `ingest_source_url` update mode behavior from the v0.2 line: missing `source.mode` defaults to `create`, duplicate creates reject, `source.mode: "update"` re-ingests an existing `source.url`, and mismatched `path_hint` or `asset_path_hint` values conflict without writing extra docs or assets.
- Preserved same-SHA source URL update behavior: same-SHA updates are no-ops without `source_updated` provenance or projection invalidation churn.
- Preserved changed-PDF source URL update behavior: changed-PDF updates preserve source and asset paths, refresh searchable extracted text and citations, emit previous/new SHA provenance, and expose stale dependent synthesis through `projection_states`.
