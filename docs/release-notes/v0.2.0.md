# OpenClerk v0.2.0

This release tightens the installed OpenClerk runner and skill contract after the first public release, with clearer routine knowledge-task policy, database-anchored vault configuration, and release-gate evidence for the current AgentOps workflow.

## Changed

- Renamed and clarified the OpenClerk skill instructions so routine knowledge requests use the installed `openclerk document` and `openclerk retrieval` runner surface, ask once for missing required fields, and reject unsupported lower-level bypasses before tools.
- Added support for root-relative notes vault paths while keeping the runner-facing document and retrieval actions stable.
- Anchored vault configuration in the SQLite database so `OPENCLERK_DATABASE_PATH` remains the routine storage anchor for data, database state, vault root, and layout conventions.
- Updated AgentOps eval coverage and sanitized real-vault evidence around installed-runner workflows, missing-field clarification, invalid-limit rejection, and lower-level bypass rejection.
