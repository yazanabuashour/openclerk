# OpenClerk v0.2.11

This patch release adds explicit retrieval regression capture/replay, search diagnostics, and maintenance posture reports while preserving local-first runner boundaries and unchanged default lexical search behavior.

## Changed

- Added `retrieval_eval_capture` as an explicit local-only retrieval regression capture action that records sanitized query/action/filter/result references, provider posture, and latency without raw vault content.
- Added `retrieval_eval_replay` for replaying captured retrieval cases and reporting Jaccard, top-1, and latency metrics for reviewable regression checks.
- Added `search_diagnostics_report` for read-only search-vs-`semantic_search` guidance with visible filters, module readiness, cost/latency posture, and no default ranking change.
- Added `maintenance_report` for read-only layout, projection freshness, relationship context, duplicate risk, module posture, and git lifecycle posture packaging without repair, cron, background jobs, or durable writes.
- Refreshed graph relationship, graph maintenance, public-vault, real-vault, and routine UX evidence after the retrieval maintenance additions.
- Updated `openclerk capabilities`, `openclerk retrieval --help`, `README.md`, and `skills/openclerk/SKILL.md` to advertise the new narrow retrieval-side surfaces and their safety boundaries.
- Preserved `ingest_source_url` update mode behavior from the v0.2 line: missing `source.mode` defaults to `create`, duplicate creates reject, `source.mode: "update"` re-ingests an existing `source.url`, and mismatched `path_hint` or `asset_path_hint` values conflict without writing extra docs or assets.
- Preserved same-SHA source URL update behavior: same-SHA updates are no-ops without `source_updated` provenance or projection invalidation churn.
- Preserved changed-PDF source URL update behavior: changed-PDF updates preserve source and asset paths, refresh searchable extracted text and citations, emit previous/new SHA provenance, and expose stale dependent synthesis through `projection_states`.
